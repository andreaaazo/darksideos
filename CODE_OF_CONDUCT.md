## Purpose

We are committed to maintaining a professional, respectful, and constructive engineering environment.

This Code of Conduct defines the expected standards of behavior for everyone contributing to this repository, including maintainers, reviewers, and collaborators.

## Expected Behavior

All contributors are expected to:

- communicate professionally and respectfully
- provide constructive, actionable feedback during code review
- assume good intent and collaborate in good faith
- focus discussions on technical outcomes and project goals
- document decisions clearly when they affect the team
- respect ownership boundaries, repository rules, and review processes

## Unacceptable Behavior

The following behavior is not acceptable in this repository or related collaboration spaces:

- harassment, discrimination, or abusive language
- personal attacks or hostile review comments
- dismissive, aggressive, or intentionally disruptive behavior
- knowingly bypassing repository rules or review requirements
- sharing confidential, proprietary, or sensitive information without authorization
- repeated off-topic or unproductive communication that disrupts engineering work

## Review and Collaboration Standards

When reviewing or discussing changes:

- keep feedback specific, technical, and actionable
- avoid vague, dismissive, or non-constructive criticism
- explain blocking concerns clearly
- separate subjective preferences from required changes
- escalate disagreements respectfully through the appropriate channels

## Responsibilities of Maintainers

Maintainers and reviewers are responsible for:

- enforcing repository rules consistently
- applying review standards fairly
- helping contributors understand required changes
- maintaining a respectful and efficient contribution process

## Reporting Concerns

If you experience or observe behavior that violates this Code of Conduct, report it privately to the maintainers or the appropriate internal engineering leadership channel.

For security-related concerns, follow the process described in [`SECURITY.md`](./SECURITY.md).

## Enforcement

Repository maintainers may take appropriate action in response to behavior that violates this Code of Conduct, including moderation of discussions, limiting collaboration access, or escalating the issue internally when required.

## Scope

This Code of Conduct applies to:

- pull requests
- issues
- code review discussions
- repository-related engineering communication
- other collaboration spaces directly related to this project
